# Gate 6 브라우저 실제 화면 QA 체크리스트

- 기준일: 2026-05-15
- 원칙: 실제 브라우저 화면 QA는 사용자가 직접 수행합니다. Codex는 이후 정적 QA, 빌드, 데이터 readback, 보안 스크립트까지만 자동 수행합니다.
- 공통 조건: 같은 계정, 같은 기준시점(2026년 4월), 같은 필터, 같은 화면 배율에서 확인합니다.

## 1. 업무 로그 메인

- 최상단 제목이 `물류센터 워크 플랫폼`인지 확인합니다.
- 통합 검색이 사용자 이름 영역과 담당 자산 영역 사이에 한 줄로 배치됐는지 확인합니다.
- 검색창에서 자산명/임차인명/계약/이슈를 검색했을 때 지도 중심이 아니라 KPI, 계약, 임차인, source 정보 중심 팝업이 열리는지 확인합니다.
- 사용자 이름 옆 프로필 영역에 사진이 있는 계정은 사진이 보이고, 사진이 없을 때만 이니셜 fallback이 보이는지 확인합니다.
- 담당 자산 블록의 자산명이 좌측 정렬이고, 긴 자산명은 말줄임표가 아니라 어절 단위 줄바꿈으로 카드 안에 들어가는지 확인합니다.
- `오늘 봐야 할 업무 신호`가 없는지 확인합니다.
- 주요 Task 수정/삭제/완료 버튼 클릭 시 확인 팝업이 뜨고, `예`를 눌러야만 동작하는지 확인합니다.
- `원본 데이터 수정` 컴포넌트가 메인 페이지 하단에 없는지 확인합니다.

## 2. Weekly

- 주간업무보고자료 업로드 팝업에 제목, 반영 기간, 파일 선택, `데이터 반영` 버튼만 보이는지 확인합니다.
- 연도, 월, 주차, 보고일, 총 자산 수, 총 연면적 블록이 한 줄에 같은 높이/너비 느낌으로 정렬됐는지 확인합니다.
- 2026년 4월 5주차가 `2026-04-27 ~ 2026-05-03`, 2026년 5월 1주차가 `2026-05-04 ~ 2026-05-10`으로 서로 겹치지 않는지 확인합니다.
- 신규 투자 Projects, 관리 Projects의 `/` 구분 내용이 bullet 중심 줄바꿈으로 보이는지 확인합니다.

## 3. Home

- 상단 KPI 월 임관리비와 월 임관리비 비중 도넛 총액이 같은 2026년 4월 기준으로 맞는지 확인합니다.
- 포트폴리오 위치 우측 표에서 부국물류센터 E. NOC가 빈값이 아니라 약 27,450원 수준으로 보이고, 저온창고 비율은 0.0%로 보이는지 확인합니다.
- 아레나스안성 E. NOC는 Data Quality에서 최신 임대료/관리비 history 미연결 사유로 확인되는지 확인합니다.
- 용도별 비율 도넛 hover 시 커서 오른쪽에 해당 조각 하나의 값만 뜨는지 확인합니다.
- 용도별 비율 범례가 상온창고, 저온창고, 복합, 사무실로만 보이고 색상이 지정 기준과 맞는지 확인합니다.
- 계약 이력 기준 임대료 추이 차트가 전체 너비를 쓰고 축명, 축 tick, 범례가 겹치지 않는지 확인합니다.
- 계약 이력 기준 임대료 추이 차트의 오른쪽 Y축 색상과 축 제목이 자산 수 series와 같은 보라색인지 확인합니다.
- 계약 이력 기준 임대료 추이 차트의 각 월 point에 hover했을 때 `신규 편입 자산`이 뜨고, `원본 표 보기`에서도 같은 월의 신규 편입 자산명이 보이는지 확인합니다.
- 만기 집중도 차트의 x/y축 글자가 잘 보이고 y축이 평 단위의 딱 떨어지는 값인지 확인합니다.
- 만기 집중도 차트가 두 개 모두 꺾은선이 아니라, 만기 임대면적은 막대, 만기 임차인 수는 꺾은선으로 색이 구분되는지 확인합니다.
- Top 자산 테이블의 핵심 임차인 컬럼과 컬럼 폭을 확인합니다.
- 임차인 계약 전체 표 보기 팝업이 충분히 넓고, 자산/임차인 필터와 sticky 컬럼이 동작하는지 확인합니다.

## 4. Asset

- KPI가 임대율, 총 임대면적, 공실면적, 월 임관리비 총액, E. NOC, 현재 임차인 수 순서인지 확인합니다.
- E. NOC가 `46,984원`처럼 원 단위로 보이는지 확인합니다.
- 임차인 현황에 E. NOC 컬럼이 평당 임대료 앞에 있고 층/세부구역 컬럼이 과하게 넓지 않은지 확인합니다.
- 면적 구성에서 임대면적/공실면적이 빠지고, 전체 연면적/전용/공용 subtotal과 비율이 연면적 기준인지 확인합니다.
- 만기 스냅샷 hover 시 구역(층), 만기일, 잔여개월이 보이는지 확인합니다.

## 5. Company

- 회사 선택을 바꾸면 KPI, 임차 자산 현황, 지도, DART, 자산별 노출도가 모두 바뀌는지 확인합니다.
- 상단의 대상/기준시점/계약/금액/DART/지도 블록이 없는지 확인합니다.
- 회사별 임차 자산 지도와 DART 상세 정보가 3.5:6.5 비율 정도인지 확인합니다.
- DART 상세 정보에서 삭제 요청 항목들이 보이지 않는지 확인합니다.
- 임차 자산 현황 테이블의 컬럼 폭과 줄바꿈이 깨지지 않는지 확인합니다.

## 6. Analysis Tools / Data Playground / Data Quality

- 대시보드 우측 상단의 `원본 데이터 수정` 버튼이 `물류센터 워크 플랫폼` 버튼 왼쪽에 있고, 클릭하면 팝업으로 `Excel 다운로드`, `수정 Excel 업로드`만 중심으로 보이는지 확인합니다.
- Data Quality 권한이 없는 사용자에게는 대시보드 우측 상단 `원본 데이터 수정` 버튼이 보이지 않는지 확인합니다.
- Analysis Tools에서 자산/기업 다중 선택, 벤치마크, 선택 계약 원장이 동작하는지 확인합니다.
- Analysis Tools 선택 자산·기업 비교에서 기업 선택 `-` 옵션이 보이지 않고, 비교 지표에서 평당 월임대료/평당 월관리비를 선택할 수 있는지 확인합니다.
- Data Playground에서 Sandbox, Explorer, BI Workspace 모드와 행/열/값/보조값/필터/Top N 피벗 조합이 동작하는지 확인합니다.
- Data Playground에서 피벗 결과 테이블이 차트보다 위에 있고, 값 필드에서 평당 월임대료/평당 월관리비를 선택할 수 있는지 확인합니다.
- Data Quality 탭이 기획추진센터 허용 인원에게만 보이는지 확인합니다.
- Data Quality 수정 요청 팝업에서 Excel처럼 before/after 값을 보고 요청할 수 있는지 확인합니다.
- 수정 Excel 업로드 후 Data Quality 최상단에 업로드한 사람, 자산 범위, 업로드 시각, 수정 행, 제외 행, 파일명, 상태가 보이는지 확인합니다.
- 승인 대기 목록에서 readback/승인/반려 버튼이 보이고 권한 없는 사용자에게는 막히는지 확인합니다.

## 7. Live 확인

- live URL에서 local과 같은 화면이 뜨는지 확인합니다.
- console error, network error, secret 노출 문자열이 없는지 확인합니다.
- Reader 권한 사용자에서 write/sync/delete/Data Quality가 차단되는지 확인합니다.
